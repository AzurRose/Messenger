# Messenger 
Программная система для обмена сообщениями и файлами по TCP с WPF-клиентом.

## Состав проекта
- `TcpMessenger.Shared` — общий протокол, типы сообщений, сериализация и фрейминг.
- `TcpMessenger.Server` — TCP-сервер для обработки подключений, сообщений, пользователей и файлов.
- `TcpMessenger.Client` — WPF-клиент, реализованный по MVVM.

## Используемые технологии
- C# / .NET 8
- WPF
- System.Net.Sockets
- async/await
- JSON-сериализация
- MVVM
- ICommand
- ObservableCollection
- IDataErrorInfo

## Возможности

Чат:
- подключение пользователя к серверу;
- отправка сообщений всем пользователям;
- отправка личных сообщений выбранному пользователю;
- отображение истории сообщений в рамках текущей сессии;
- отображение списка активных пользователей.

Файлы:
- выбор файла на клиенте;
- загрузка файла на сервер;
- уведомление клиентов о доступном файле;
- скачивание выбранного файла с сервера;
- отображение прогресса загрузки и скачивания.

Сервер:
- одновременная работа нескольких клиентов;
- асинхронная обработка подключений;
- маршрутизация broadcast-сообщений;
- маршрутизация private-сообщений;
- управление пользовательскими сессиями;
- хранение загруженных файлов в папке `ServerStorage`.

## Протокол взаимодействия

Обмен данными выполняется через TCP. Каждое сообщение передаётся в формате:

```text
[4 байта длины JSON][JSON-сообщение]
```

Такой формат позволяет корректно отделять одно сообщение от другого в TCP-потоке.

Основной объект протокола — `NetworkMessage`.

Поддерживаемые типы сообщений:

- `Login`
- `Chat`
- `PrivateChat`
- `UserList`
- `System`
- `FileUploadStart`
- `FileUploadChunk`
- `FileAvailable`
- `FileDownloadRequest`
- `FileDownloadStart`
- `FileDownloadChunk`
- `Error`
- `Disconnect`

Файлы передаются частями. Содержимое каждой части кодируется в Base64 и помещается в JSON-сообщение типа `FileUploadChunk` или `FileDownloadChunk`.

## Как запустить
1. Открыть `TcpMessengerLab.sln` в VS Code.
2. Запустить проект `TcpMessenger.Server`.
3. Запустить один или несколько экземпляров `TcpMessenger.Client`.
4. В клиентах указать:
   - Host: `127.0.0.1`
   - Port: `5000`
   - разные имена пользователей.
5. Нажать `Connect`.
6. Проверить общий чат, личные сообщения и передачу файлов.

## Архитектурa

- Сетевой протокол вынесен в отдельный общий проект.
- Сервер не зависит от WPF-клиента.
- Клиентская логика вынесена в `MainViewModel` и `TcpClientService`.
- UI связан с ViewModel через Binding.
- Команды реализованы через `ICommand`.
- Коллекции сообщений, пользователей и файлов реализованы через `ObservableCollection`.
- Обновление UI выполняется через Dispatcher.
- Передача данных реализована асинхронно.
